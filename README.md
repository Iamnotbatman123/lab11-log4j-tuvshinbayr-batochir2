# **BankAccount**
## Үндсэн функцууд

Уг класс нь дараах банкны үйлдлүүдийг багтаасан бөгөөд оролт бүрийг шалгах зохицуулалттай:

    Байгуулагч (Constructor): Дансыг эхлүүлнэ. Хэрэв хасах утгатай үлдэгдэл оруулбал 0 болгон тохируулж, анхааруулга (warning) лог хөтөлнө.

    Орлого (Deposit): Дансны үлдэгдлийг нэмэгдүүлнэ. Хасах утгатай дүн оруулахыг хориглож, үлдэгдлийн өөрчлөлтийг тэмдэглэнэ.

    Зарлага (Withdraw): Данснаас зарлага гаргана. Үлдэгдэл хүрэлцэхгүй тохиолдолд гүйлгээг цуцалж, алдааны (error) лог хөтөлнө.

    Үлдэгдэл шалгах: Одоогийн үлдэгдлийг буцаана.

    Ноцтой алдааны симуляци: Системийн ноцтой эвдрэлийг илэрхийлэх FATAL түвшний лог үүсгэх функцийг агуулсан.

## Лог хөтлөлт (Logging Implementation)

Энэхүү кодын гол онцлог нь Log4j2-ын түвшингүүдийг ашиглан програмын төлөв байдлыг нарийвчлан хянах боломжийг олгосонд оршино:
Түвшин	Ашиглалт
TRACE	Функц эхлэх болон дуусах цэг бүрийг тэмдэглэнэ.
DEBUG	Гүйлгээний өмнөх болон дараах үлдэгдэл, төлөв байдлыг бүртгэнэ.
INFO	Амжилттай болсон орлого, зарлагын гүйлгээг бүртгэнэ.
WARN	Буруу оролт (жишээ нь: хасах утгатай мөнгөн дүн) өгөх үед анхааруулна.
ERROR	Дансны үлдэгдэл хүрэлцэхгүй үед алдааг мэдээлнэ.
FATAL	Системийн хэмжээний засаж боломгүй ноцтой алдааг тэмдэглэнэ.

# **Customer**
## Үндсэн функцууд

Уг класс нь хэрэглэгчийн бүртгэл болон и-мэйл хаягтай ажиллах дараах боломжуудтай:

    Байгуулагч (Constructor): Хэрэглэгчийн нэр болон и-мэйлийг тохируулна. И-мэйл хаягийг бүртгэхдээ нууцлалын үүднээс тодорхой хэсгийг нь mask буюу далдлаж логддог.

    И-мэйл далдлах (Masking): Хэрэглэгчийн хувийн мэдээллийг хамгаалах зорилгоор и-мэйл хаягийн дунд хэсгийг одоор (***) орлуулж харуулна.

    Домэйн авах (getDomain): И-мэйл хаягаас домэйн нэрийг (жишээ нь: @GMAIL.COM) салгаж авна. Хэрэв и-мэйл хаяг байхгүй (null) бол алдааг бүртгэж, "TRY AGAIN" гэсэн утга буцаана.

## Лог хөтлөлт (Logging Implementation)

Энэхүү кодод логийг мэдээллийн аюулгүй байдал болон логик алдааг илрүүлэхэд ашигласан:
Түвшин	Ашиглалт
TRACE	getDomain() функц руу нэвтэрч буйг нарийвчлан хянана.
DEBUG	Хэрэглэгч үүсгэх үед болон и-мэйл салгах оролдлого бүрийг тэмдэглэнэ.
ERROR	И-мэйл хаяг null байх эсвэл домэйн салгах үйлдэл амжилтгүй болох үед бүртгэнэ.
## Онцлох шинж чанарууд

    Мэдээллийн нууцлал (Data Privacy): mask() функцийг ашиглан лог файл руу хэрэглэгчийн хувийн и-мэйлийг ил задгай бичихээс сэргийлсэн.

    Алдаа шалгалт: И-мэйл хаяг хоосон байх үед үүсэж болох NullPointerException-оос сэргийлж, алдааны түвшинд лог хөтлөлт хийсэн.
# **Log4j2 тохируулга**
## Log4j2 Тохиргооны Тайлбар

Уг тохиргоо нь програмын явц дахь мэдээллийг консол (Console) болон файл руу (RollingFile) нэгэн зэрэг бүртгэх зорилготой.
### 1. Appenders (Мэдээлэл дамжуулагчид)

Лог мэдээллийг хаашаа илгээхийг энд тодорхойлсон:

    Console Appender: Логуудыг SYSTEM_OUT буюу стандарт консол дээр хэвлэнэ. Хөгжүүлэлтийн явцад алдааг шууд харахад тустай.

    RollingFile Appender: Логуудыг logs/app.log файлд хадгална.

        Давуу тал: Файл нь өдөр бүр шинэчлэгдэж (logs/app-yyyy-MM-dd.log), хуучин логуудыг архивлах бодлогоор ажиллана.

        TimeBasedTriggeringPolicy: Өдөр бүр (interval="1") шинэ файл үүсгэхээр тохируулсан.

### 2. Pattern Layout (Хэвлэх хэлбэр)

Лог мэдээлэл ямар бүтэцтэй харагдахыг дараах загвараар тохируулсан:
%d{yyyy-MM-dd HH:mm:ss} [%t] %-5level %c{1}:%L - %msg%n

    %d: Огноо болон цаг.

    %t: Гүйцэтгэж буй thread-ийн нэр.

    %-5level: Лог түвшин (INFO, WARN, ERROR гэх мэт, 5 тэмдэгтийн зайтай).

    %c{1}:%L: Классын нэр болон кодын мөрийн дугаар.

    %msg%n: Мессеж болон шинэ мөр.

## Loggers (Лог хөтлөгч)

Лог хөтлөлтийн үндсэн дүрмийг энд тодорхойлно:

    Root Level: info түвшнээс дээш (INFO, WARN, ERROR, FATAL) бүх мэдээллийг бүртгэнэ.

    Холболт: Үүсгэсэн Console болон RollingFile гэсэн хоёр appender-ийг хоёуланг нь идэвхжүүлсэн.

## Ашиглах заавар

    Уг файлыг төслийн src/main/resources хавтсанд байршуулна.

    Програмыг ажиллуулахад logs/ хавтас автоматаар үүсэж, дотор нь лог файлууд бичигдэж эхэлнэ.

    Хэрэв та илүү нарийвчилсан лог (TRACE, DEBUG) харахыг хүсвэл <Root level="info">-г <Root level="all"> болгож өөрчлөх боломжтой.

### Төслийн бүтэц (Жишээ)
Plaintext

project/
├── src/
│   └── main/
│       └── resources/
│           └── log4j2.xml
└── logs/
    └── app.log (автоматаар үүснэ)
